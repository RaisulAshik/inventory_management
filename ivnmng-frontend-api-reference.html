Axios Base Setup (Frontend)

import axios from 'axios';

const api = axios.create({
  baseURL: 'http://localhost:3000/api/v1',
  headers: { 'Content-Type': 'application/json' },
});

// Request interceptor — attach token + tenant ID
api.interceptors.request.use((config) => {
  const token = localStorage.getItem('accessToken');
  const tenantId = localStorage.getItem('tenantId');
  if (token)    config.headers['Authorization'] = `Bearer ${token}`;
  if (tenantId) config.headers['X-Tenant-ID'] = tenantId;
  return config;
});

// Response interceptor — auto-refresh token on 401
api.interceptors.response.use(
  (res) => res,
  async (err) => {
    if (err.response?.status === 401) {
      const refreshToken = localStorage.getItem('refreshToken');
      if (refreshToken) {
        const { data } = await axios.post('/api/v1/auth/refresh', { refreshToken });
        localStorage.setItem('accessToken', data.data.accessToken);
        err.config.headers['Authorization'] = `Bearer ${data.data.accessToken}`;
        return api(err.config);
      }
      // Redirect to login
      window.location.href = '/login';
    }
    return Promise.reject(err);
  }
);

2. Authentication Flow

Tenant User Flow

1. POST /api/v1/auth/login  →  { accessToken, refreshToken, user, tenantId }
2. Store:  localStorage.setItem('accessToken', accessToken)
           localStorage.setItem('refreshToken', refreshToken)
           localStorage.setItem('tenantId', user.tenantId)
3. All subsequent requests must include:
   - Header: Authorization: Bearer {accessToken}
   - Header: X-Tenant-ID: {tenantId}
4. On 401 → POST /api/v1/auth/refresh  →  new tokens
5. POST /api/v1/auth/logout  →  clear localStorage

Admin (SaaS) Flow

1. POST /api/v1/admin/auth/login  →  { accessToken, refreshToken, admin }
2. Store:  localStorage.setItem('adminToken', accessToken)
3. All admin requests:
   - Header: Authorization: Bearer {adminToken}
   - NO X-Tenant-ID needed for admin routes

3. Required Request Headers

HeaderValueRequired For
AuthorizationBearer {accessToken}All protected routes
X-Tenant-ID{tenantId} (UUID)All tenant routes (NOT admin/master)
Content-Typeapplication/jsonPOST / PATCH / PUT requests
Note: Admin routes (/admin/*) and master routes (/master/*) do NOT require X-Tenant-ID. Public routes (/auth/login, /auth/register, /health) do not require Authorization.

5. Error Handling

{
  "success": false,
  "statusCode": 400,
  "message": "Validation failed",
  "errors": [
    { "field": "email", "message": "email must be a valid email" }
  ]
}
Status CodeMeaningFrontend Action
400Bad Request / Validation ErrorShow field errors to user
401Unauthorized (invalid/expired token)Refresh token or redirect to login
403Forbidden (no permission)Show "Access Denied" message
404Resource Not FoundShow "Not Found" message
409Conflict (duplicate record)Show conflict message
422Unprocessable EntityShow business rule violation
429Too Many Requests (rate limit)Show "Please wait" / retry after delay
500Internal Server ErrorShow generic error, log to monitoring
